ИНСТРУКЦИЯ: генерация оглавления лекции на основе JSON-транскрипта
===================================================================

ВХОДНЫЕ ДАННЫЕ
--------------
- filename_dg.json — транскрипт Deepgram (параграфы с таймкодами и текстом)

ВЫХОДНОЙ ФОРМАТ
---------------
Иерархический список заголовков h1/h2/h3:

  <h1>Название лекции</h1> §N [ЧЧ:ММ:СС]
  <h2>Название раздела</h2> §N [ЧЧ:ММ:СС]
    <h3>Название подраздела</h3> §N [ЧЧ:ММ:СС]

где §N — индекс параграфа JSON (с нуля), с которого начинается тема.

ВЫХОДНОЙ ФАЙЛ
-------------
Сохранить результат в текстовый файл с именем:
  <имя_входного_json>_toc.txt
Пример: входной файл 01_vvip_dg.json → выходной файл 01_vvip_dg_toc.txt

ШАГ 1. ИЗВЛЕЧЬ ПАРАГРАФЫ ИЗ JSON
----------------------------------
  import json

  data = json.load(open('filename_dg.json'))[0]
  paras = data['results']['channels'][0]['alternatives'][0]['paragraphs']['paragraphs']

  def fmt(s):
      t = int(s)
      return f'{t//3600:02d}:{(t%3600)//60:02d}:{t%60:02d}'

  for i, p in enumerate(paras):
      text = ' '.join(s['text'] for s in p['sentences'])
      print(f'§{i} [{fmt(p["start"])}] {text[:100]}')

Зафиксировать:
  - Общую длительность (start последнего параграфа → расчёт кол-ва заголовков)
  - Индекс §0 и его таймкод — это фактическое начало речи (может быть не 00:00:00)


ШАГ 2. РАССЧИТАТЬ ДОПУСТИМОЕ КОЛИЧЕСТВО ЗАГОЛОВКОВ
----------------------------------------------------
По длительности видео:

  30 минут  →  8–10 заголовков
  60 минут  → 17–20 заголовков
  90 минут  → 25–30 заголовков
  120 минут → 35–40 заголовков

Это МАКСИМУМ. Лучше меньше, чем больше.
Средний шаг между заголовками: строго 3–5 минут.

Исключение по плотности: h3 сразу после h2 может иметь тот же таймкод
или отставать не более чем на 1 минуту (короткое вступление к разделу).


ШАГ 3. ОПРЕДЕЛИТЬ КРУПНЫЕ СМЫСЛОВЫЕ БЛОКИ
-------------------------------------------
Читать параграфы и отмечать переходы между БОЛЬШИМИ темами.

✅ Выделять в отдельный заголовок:
  - Переход к новой крупной теме
  - Разбор конкретного текста (шлока, сутра, цитата)
  - Развёрнутый ответ на вопрос слушателя (см. правило ниже)
  - Практическое упражнение
  - Смена философской школы или концепции

❌ НЕ выделять:
  - Каждое упоминание нового термина внутри одной темы
  - Короткие отступления и примеры (менее 2–3 минут)
  - Перечисление связанных понятий в рамках одного сюжета
  - Если 5 терминов упомянуты подряд в одном контексте → это ОДИН заголовок
  - Вступление и обзор литературы как два отдельных h2, если они занимают
    менее 5 минут суммарно → объединить в один h2 «Вступление. Литература…»
    Разбивать на два допустимо только если блок литературы занимает ≥ 5 минут
    самостоятельно.

Правило для длинных h2-разделов:
  Если h2-раздел длится более 7 минут и содержит 2 или более различимые
  подтемы → обязательно добавить h3 для каждой подтемы.
  Монолитный h2 без h3 при длине > 7 минут — ошибка структуры.

Правило для блока вопросов слушателей:
  Если блок «Вопросы слушателей» длится более 3 минут и содержит 2 или более
  отдельных вопроса → каждый вопрос оформляется как h3 с кратким содержанием.
  Пример: «О природе сознания», «О связи санкхьи и йоги».
  Если вопрос один и занимает менее 3 минут → h3 не нужен.


ШАГ 4. СФОРМИРОВАТЬ ЧЕРНОВИК ЗАГОЛОВКОВ
-----------------------------------------
Для каждого заголовка:
  - Уровень: h1 (название лекции), h2 (раздел), h3 (подраздел)
  - Длина: 2–7 слов
  - Содержание: конкретная тема / философ / текст / вопрос
  - Примеры: «Понятие дхармы», «Аргументация Шанкары»,
             «Разбор шлоки 2.47», «Вопрос о медитации»

h1 всегда ставится на §0 — первый параграф JSON, независимо от того,
говорит в нём лектор или ведущий.
Текст h1 отражает название лекции (берётся из содержания, которое излагает лектор).
h1 — единственный на всю лекцию.
❌ Недопустимо: h1 на §1 или позже, даже если §0 — слова ведущего, а не лектора.

Правило содержания заголовка:
  Заголовок отражает ГЛАВНУЮ тему параграфа, а не побочную.
  При сомнении — брать тему первого предложения §N.
  ❌ Недопустимо заменять санскритский термин описательным оборотом,
  если этот термин явно назван лектором.
  Пример: если лектор говорит «Арьяварта» → писать «Арьяварта»,
  а не «региональные идентичности».


ШАГ 5. ПРИВЯЗАТЬ ЗАГОЛОВКИ К ПАРАГРАФАМ
-----------------------------------------
Таймкод заголовка должен точно совпадать со start одного из параграфов JSON.

Алгоритм:
  1. Определить момент смены темы по содержанию
  2. Найти параграф §N, где start >= этого момента
  3. Убедиться, что первое предложение §N действительно начинает нужную тему
  4. Если первое предложение §N ещё о предыдущей теме → сдвинуть на §N+1
  5. Дополнительно проверить §N-1: если новая тема уже началась в его последних
     предложениях → рассмотреть §N-1 как кандидата, при условии что интервал
     с предыдущим заголовком остаётся ≥ 3 минут
  6. Использовать именно этот §N и его таймкод

Это гарантирует, что заголовок будет вставлен между параграфами,
а не в середину одного из них.


ПРАВИЛА ТЕРМИНОЛОГИИ
---------------------
Исправлять ошибки транскрипции в названиях заголовков:
  - Бхагавад-гита (не «Пхагавд-гита», не «Бхагавадгита»)
  - йога-сутры (не «йогасутры»)
  - сандхи (не «сантхи»)
  - санграха (не «самграха») — транслит. saṃgraha
  - аджи́вика (не «аджибика») — ударение на второй слог
  - Арьяварта (не «арьяварта») — с заглавной буквы как топоним
  - Проверять орфографию санскритских терминов перед финальной выдачей

❌ Не заменять термины описательными фразами:
  Неверно: «региональные идентичности» → верно: «Арьяварта»
  Неверно: «цель даршаны» → верно: «цель философии» (если лектор использует
  именно это слово в данном контексте)


ПРОВЕРКИ ПЕРЕД ФИНАЛЬНОЙ ВЫДАЧЕЙ
==================================

□ ЧЕК 1. Количество заголовков в норме
     Подсчитать все h2 + h3 (h1 не считать).
     Сравнить с допустимым диапазоном для данной длительности (см. Шаг 2).
     Если больше максимума → объединить мелкие смежные заголовки.

□ ЧЕК 2. Таймкоды совпадают с параграфами
     Для каждого заголовка: его [ЧЧ:ММ:СС] должен совпадать со start
     какого-либо параграфа из JSON. Не интерполировать, не округлять.

□ ЧЕК 3. Первое предложение §N соответствует теме заголовка
     Прочитать первое предложение каждого §N.
     Убедиться, что оно начинает нужную тему, а не заканчивает предыдущую.
     Если нет → сдвинуть на §N+1 и повторить проверку.
     Также проверить §N-1: если новая тема уже началась в его последних
     предложениях → рассмотреть §N-1 вместо §N (при соблюдении интервала ≥ 3 мин).
     После сдвига убедиться, что заголовок не оказался позже следующего
     по списку и интервал между соседними заголовками остался ≥ 3 минут.
     Если интервал нарушается → объединить два заголовка в один.

□ ЧЕК 4. Интервалы между заголовками
     Средний шаг между соседними заголовками: 3–5 минут.
     Исключение: h3 сразу после h2 — допустимо совпадение или разрыв ≤ 1 мин.
     Если два заголовка стоят ближе 3 минут друг к другу (и это не h3 после h2)
     → объединить.

□ ЧЕК 5. Иерархия корректна
     h3 не стоит без предшествующего h2.
     h1 ровно один, в самом начале.
     h3 логически является подтемой ближайшего h2 выше.

□ ЧЕК 6. Длинные разделы не остались без подзаголовков
     Проверить каждый h2 без h3: если его длина > 7 минут → найти подтемы
     и добавить h3. Монолитный раздел > 7 минут без h3 — ошибка.

□ ЧЕК 7. Вопросы слушателей детализированы
     Если блок вопросов > 3 минут и содержит 2+ вопроса → должны быть h3.
     Одиночный заголовок «Вопросы слушателей» без h3 при таких условиях —
     ошибка.

□ ЧЕК 8. Заголовки не содержат описательных замен терминов
     Убедиться, что санскритские термины и топонимы написаны точно,
     а не заменены общими фразами (см. Правила терминологии).


ТИПИЧНЫЕ ОШИБКИ
---------------
- Заголовок поставлен на §N, где лектор ещё заканчивает предыдущую тему → чек 3
- Слишком много заголовков: каждый термин выделен отдельно → чек 1
- Таймкод не совпадает ни с одним параграфом (взят «на глаз») → чек 2
- Два заголовка стоят с разрывом менее 3 минут → чек 4
- h3 висит без h2 выше → чек 5
- Санскритский термин написан с ошибкой транскрипции → правила терминологии
- §N выбран вместо §N+1: тема уже началась в конце предыдущего параграфа → чек 3
- §N+1 выбран вместо §N-1: тема началась ещё в предпоследнем параграфе → чек 3
- h1 поставлен на §1 вместо §0, так как §0 — слова ведущего → шаг 4
- h3 выделен на одиночный параграф-перечисление без самостоятельного смысла → чек 1
- Вступление и литература разбиты на два h2 при суммарной длине < 5 минут → шаг 3
- h2-раздел длиннее 7 минут оставлен без h3 → чек 6
- Блок вопросов > 3 минут оставлен без h3 → чек 7
- Санскритский термин заменён описательной фразой → чек 8, правила терминологии
- «самграха» вместо «санграха» в заголовке или в примере → правила терминологии
