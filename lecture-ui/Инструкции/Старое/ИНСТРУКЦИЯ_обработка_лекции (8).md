Этот документ — полная инструкция для ИИ-ассистента. Прикрепи его в начало нового чата вместе с материалами лекции.

## Быстрая шпаргалка: что сделать перед стартом

**Шаг А — прикрепить файлы** (в таком порядке):
1. Стенограмма новой лекции — `.txt`
2. Презентация новой лекции — `.pptx` *(если есть; если нет — слайды пропускаются)*
3. HTML предыдущей лекции — например `01.html` (образец вёрстки)
4. Субтитры новой лекции — `.srt`
5. Этот файл инструкции — `ИНСТРУКЦИЯ_обработка_лекции.md`

**Шаг Б — написать в чат:**
- ID ролика YouTube (11 символов из `youtube.com/watch?v=XXXXXXXXXXX`)
- Полный URL ролика Rutube (нужен для embed и тайм-кодов)

**Шаг В — дать команду:**
> Обработай лекцию по инструкции

## Что нужно приложить к чату

Прикрепи четыре файла **в таком порядке**, а затем напиши: «Обработай лекцию по инструкции»:

1. **Стенограмма лекции** — текстовый файл `.txt` с расшифровкой речи лектора (уже отредактированный, без слов-паразитов и технических артефактов).
2. **Презентация** — файл `.pptx` с иллюстративными слайдами к лекции *(необязательно: если презентации нет, шаги 2 и 4 по расстановке слайдов пропускаются целиком)*.
3. **Файл разметки** — готовый HTML-файл предыдущей лекции серии, например `01.html` (служит образцом структуры, стилей, JS-блока и оглавления — **не перезаписывать**). Прикладывать обязательно: он гарантирует точное визуальное совпадение между лекциями.
4. **Файл субтитров** — файл `.srt` с автоматическими субтитрами YouTube (нужен для временны́х меток).

## Что должен сделать ассистент

### Шаг 0 — Редактура стенограммы

Стенограмма может быть как уже отредактированной, так и сырой (прямо с распознавания речи). Перед созданием HTML привести её в порядок.

**Санскритизмы и термины** — главная проблема. Автоматическое распознавание речи искажает их непредсказуемо. Исправлять, опираясь на содержание слайдов презентации (там термины написаны правильно), контекст фразы и знание предмета. Типичные искажения: «Игведа» → Ригведа, «Отхрководе» → Атхарваведа, «Еджурведа» → Яджурведа, «дакшне» → дакшина, «Упонишады» → Упанишады.

**Речевые артефакты** — удалить слова-паразиты («э», «вот», «ну» в избытке), незаконченные фразы, самоперебивы, повторы одного слова подряд.

**Синтаксис** — восстановить там, где устная речь не складывается в читаемое предложение, сохраняя при этом авторский стиль и интонацию лектора.

**Диалог** — реплики слушателей в конце лекции разметить отдельно: они войдут в блок `<div class="dialog">`.

После редактуры сохранить исправленный текст — он станет основой для HTML.

### Шаг 1 — Прочитать образец

Открыть приложенный HTML-файл (`файл разметки`), изучить его структуру:
- теги `<h2 id="...">` и `<h3 id="...">` с атрибутами `id`
- блок `<nav class="toc">` — оглавление
- теги `<figure>` с `<img src="./src/img/NN_MM.jpg">` и `<figcaption>`
- класс `.yt-link` рядом с заголовками
- класс `.yt-toc` и `.rt-toc` в оглавлении
- отдельный файл CSS (`<link rel="stylesheet" href="./src/style.css">`)

Новый HTML должен полностью повторять эту структуру и стилистику. Проще всего скопировать образец и адаптировать его содержимое, чем создавать с нуля — так гарантируется побайтовое совпадение классов, JS-блока и CSS-связей.

### Шаг 3 — Разобрать стенограмму

Прочитать текстовый файл стенограммы. Выделить смысловые разделы по содержанию — они станут заголовками `<h2>`. Более мелкие темы внутри разделов — `<h3>`.

Типичная структура лекции:

### Шаг 5 — Добавить временны́е метки YouTube

#### Как читать `.srt`

Файл субтитров содержит блоки:
```
НОМЕР
ЧЧ:ММ:СС,МС --> ЧЧ:ММ:СС,МС
Текст фразы
```

Алгоритм поиска метки для раздела:
1. Найти в субтитрах фразы, соответствующие **началу** темы раздела.
2. Взять временну́ю метку **первого** такого блока.
3. Перевести в секунды: `ЧЧ×3600 + ММ×60 + СС`.
4. Сформировать ссылку: `https://youtu.be/VIDEO_ID?t=СЕКУНДЫ`.

#### Особые случаи

- Если тема кратко упомянута в начале лекции, а подробно разбирается позже — вести ссылку на **подробный** разбор.
- Если лектор говорит «об этом я уже сказал в начале» и **не разворачивает тему повторно** — вести ссылку на тот момент, когда он произносит эту фразу, а **не** на раннее упоминание. Это сохраняет хронологический порядок меток.
- Раздел «Вопросы и обсуждение» — метка на первую реплику слушателя.

#### Обязательное требование: хронологический порядок

Временны́е метки в оглавлении **должны строго возрастать** сверху вниз. Перед финальной выдачей проверить: каждая следующая метка ≥ предыдущей. Если это не так — найти ближайший момент в субтитрах, когда лектор **хотя бы упоминает** данную тему в нужном месте хронологии, и использовать его.

#### Формат ссылок в HTML

Каждая метка выводится как `YT/RT` — две ссылки через косую черту. Время (`MM:SS`) **скрыто из видимого текста** и вынесено в `title`-атрибут обеих ссылок (всплывает при наведении курсора). Разделитель `/` обёрнут в `<span class="link-sep">` — он скрывается CSS когда выбран один хостинг.

В заголовке:
```html
<h2 id="slug">Название <span class="yt-link"><a href="https://youtu.be/ID?t=SEC" title="MM:SS" target="_blank">YT</a><span class="link-sep">/</span><a href="https://rutube.ru/video/private/ID/?p=TOKEN&t=SEC" title="MM:SS" target="_blank">RT</a></span></h2>
```

В оглавлении:
```html
<a class="yt-toc" href="https://youtu.be/ID?t=SEC" title="MM:SS" target="_blank">YT</a><span class="link-sep">/</span><a class="rt-toc" href="https://rutube.ru/video/private/ID/?p=TOKEN&t=SEC" title="MM:SS" target="_blank">RT</a>
```

Rutube принимает тайм-код через параметр `&t=СЕКУНДЫ`.

Стили: `YT` — красный `#c0392b`; `RT` — синий `#1a6fa8`.

## Важные технические детали

- **Сессия не сохраняется**: каждый новый чат — чистая файловая система. Образец HTML (`NN.html`) и `src/style.css` нужно прикладывать заново. Именно поэтому HTML предыдущей лекции входит в обязательный список прикладываемых файлов — он одновременно служит и образцом вёрстки, и источником актуального JS-блока и CSS-ссылок.
- **Слайды не встраиваются в HTML**: в `<img src>` прописывается только путь `./src/img/NN_MM.jpg`. Реальные файлы изображений пользователь копирует вручную из папки с презентацией.
- **Ширина 780px**: выбрана из расчёта ~70–75 символов в строке для PT Serif 17px — оптимум классической типографики. Менять только если экран значительно шире или уже стандартного ноутбука.
- **YouTube ссылки**: формат `https://youtu.be/VIDEO_ID?t=СЕКУНДЫ` — параметр `t` в целых секундах, без миллисекунд.
- **Встроенное видео**: по умолчанию активен режим с iframe. Правая панель занимает **50% ширины экрана** и делится на две зоны: `#video-player-area` (тёмный фон, плеер 16:9, высота не более 50vh) и `#video-extra` (фон `--paper`, содержит `#toc-panel` с оглавлением и собственным скроллингом). Вся правая панель sticky и занимает 100vh — плеер сверху, оглавление растягивается и скроллится снизу. Вкладки YouTube / Rutube переключают источник.
- **Кнопка переключения режима**: `position: fixed`, правый нижний угол. Бордовая в режиме embedded, зелёная в режиме external.
- **Режим external**: однocolumn-layout, тайм-коды открывают новую вкладку как обычные ссылки. Оглавление возвращается на своё исходное место в тексте через `moveToc(false)`.
- **Перемещение TOC**: в режиме embedded оглавление (`#toc-nav`) перемещается в `#toc-panel` внутри правой панели через `appendChild`. При переключении в external — возвращается через `insertBefore`. Дублирование DOM-узла не используется.
- **Видимость ссылок YT/RT**: управляется переменной `linkMode` (`'both'` | `'yt'` | `'rt'`) и классами на `<body>`: `links-yt-only` скрывает все RT-ссылки и разделители, `links-rt-only` — все YT-ссылки. По умолчанию показываются оба (`both`). После первого клика на любую YT или RT ссылку (в тексте, оглавлении или кнопках плеера) `linkMode` фиксируется на выбранном хостинге. Переключить можно кнопками YouTube/Rutube в контейнере плеера.
- **Rutube embed**: URL плеера имеет вид `https://rutube.ru/play/embed/ВИДЕО_ID/?p=ТОКЕН`. Тайм-код добавляется как `&t=СЕКУНДЫ&autoplay=1`. Rutube игнорирует смену `src` у существующего iframe — поэтому при каждой перемотке iframe полностью пересоздаётся через `removeChild` + `createElement` (функция `reloadFrame`). После загрузки iframe отправляется `postMessage('{"method":"play"}', "https://rutube.ru")` для принудительного старта воспроизведения — браузер разрешает autoplay, так как пользователь только что кликнул.
