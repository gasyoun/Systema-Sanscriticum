ИНСТРУКЦИЯ: генерация списка figure-тегов по паре PDF (слайды) + JSON (транскрипт)
=================================================================================
(версия 4)

ВХОДНЫЕ ДАННЫЕ
--------------
- filename.pdf     — слайды лекции (каждая страница = один слайд)
- filename_dg.json — транскрипт Deepgram (параграфы с таймкодами и текстом)

ВЫХОДНОЙ ФАЙЛ
-------------
Сохранить результат в текстовый файл в кодировке UTF-8 с именем:
  <имя_входного_json>_fig.txt
Пример: входной файл 01_vvip_dg.json → выходной файл 01_vvip_dg_fig.txt

ВНАЧАЛЕ ПРОСТО ПРОВЕРИТЬ ФАЙЛЫ НА СООТВЕТСТВИЕ, может быть ситуация несоответствия PDF данной лекции из-за ошибки пользователя!

Предварительная проверка соответствия PDF и JSON:
  - Определить общую длительность транскрипта (последний таймкод) в минутах
  - Убедиться, что количество страниц PDF разумно для этой длительности
    (ориентир: ~1 слайд на 2–4 минуты лекции)
  - Если соотношение резко нарушено (например, 30 слайдов на 10 минут или
    3 слайда на 90 минут) — сообщить пользователю о возможном несоответствии
    файлов и не продолжать генерацию до подтверждения

ВЫХОДНОЙ ФОРМАТ
---------------
Одна строка на слайд:

  §N <figure><img K><figcaption>Заголовок слайда</figcaption></figure>

где:
  §N  — индекс параграфа JSON (с нуля), перед которым логически появляется слайд
  K   — порядковый номер страницы PDF (с единицы), он же номер слайда


ШАГ 1. ИЗВЛЕЧЬ ЗАГОЛОВКИ СЛАЙДОВ ИЗ PDF
-----------------------------------------
Использовать pdfplumber:

  import pdfplumber
  doc = pdfplumber.open('filename.pdf')
  for i, page in enumerate(doc.pages):
      lines = [l.strip() for l in (page.extract_text() or '').split('\n') if l.strip()]
      print(f'Стр.{i+1}: {lines[:3]}')

Для каждой страницы определить:
  - Есть ли ЯВНЫЙ заголовок (отдельная строка до буллитов/тезисов)?
  - Если да → взять его дословно как figcaption
  - Если нет (слайд начинается сразу с буллита «•» или тезиса) → придумать
    краткий описательный заголовок (2–6 слов) по содержанию слайда

Признаки ОТСУТСТВИЯ явного заголовка:
  - Первая строка начинается с пробела или буллита
  - Первая строка — это уже тезис («Аналог слова…», «Даршана – система…»)
  - Первые строки — имя собственное без пояснения («Mādhava Vidyāraṇya (XIV в.)»)

Особые случаи при извлечении заголовка:
  - Титульный слайд (стр. 1): всегда привязывается к §0, даже если лектор
    произносит название курса не в самом начале транскрипта. figcaption —
    название курса с титульного слайда.
  - Двуязычный заголовок: если на слайде присутствуют и русский, и английский
    варианты заголовка — использовать тот язык, на котором ведётся лекция.
    Если лекция на русском, а заголовок только на английском — оставить
    английский дословно, не переводить.
  - Визуально выделенный элемент: если pdfplumber возвращает несколько строк,
    претендующих на роль заголовка, приоритет у визуально выделенного элемента
    (другой шрифт, цвет, рамка), а не у первой строки при парсинге. При
    необходимости сверить с визуальным содержимым страницы PDF.
  - Одинаковый заголовок на нескольких слайдах: если pdfplumber возвращает
    один и тот же заголовок для двух или более слайдов (например, два слайда
    «Тиртхи в Махабхарате» — обложка книги и разворот с текстом) — уточнить
    figcaption по реальному содержанию каждого слайда, добавив уточняющее
    слово или подзаголовок. Не оставлять дублирующиеся figcaption.
  - «Немой» слайд: лектор показывает слайд без явного называния темы
    (пауза, «посмотрите», переход без комментария). Такой слайд получает
    §N = параграф ближайшей смены темы или паузы; figcaption придумывается
    по содержанию слайда.
  - Несколько «немых» слайдов подряд в начале лекции: если в начале идёт
    серия иллюстративных слайдов без явных комментариев (сравнение культур,
    геологический контекст, вводный визуальный ряд) — привязывать каждый к
    параграфу, где лектор разворачивает соответствующую тему, пусть даже
    кратко. Не сваливать все в §0.


ШАГ 2. ИЗВЛЕЧЬ ПАРАГРАФЫ ИЗ JSON
----------------------------------
  import json
  data = json.load(open('filename_dg.json'))[0]
  paras = data['results']['channels'][0]['alternatives'][0]['paragraphs']['paragraphs']

  for i, p in enumerate(paras):
      start = p['start']  # секунды float
      text  = ' '.join(s['text'] for s in p['sentences'])
      # → §i [ЧЧ:ММ:СС] первые 80 символов текста

Примечание: §N — индекс параграфа, начиная с 0. Первый параграф — §0, второй — §1 и т.д.

Технические параграфы: в начале онлайн-лекций типичны параграфы с
организационным диалогом («включаем презентацию», «видно или нет», «закройте
предупреждение»). Такие параграфы не содержат содержательного материала —
пропускать их при поиске привязки слайдов.


ШАГ 3. СОПОСТАВИТЬ СЛАЙД С ПАРАГРАФОМ
---------------------------------------
Для каждого слайда K найти параграф §N, перед которым лектор начинает
говорить о теме этого слайда.

Алгоритм:
  1. Определить ключевые слова/тему слайда K по его заголовку и содержанию
  2. Найти в тексте параграфов первое место, где лектор начинает разворачивать
     эту тему (объяснять, показывать, цитировать). Мимолётное упоминание темы
     в контексте другого тезиса («об этом подробнее позже») не засчитывается.
  3. Назначить §N = индекс этого параграфа

Правила:
  - §N — это параграф, который НАЧИНАЕТ тему слайда, а не продолжает
  - Если тема начинается в середине параграфа — брать этот параграф,
    не следующий
  - Два и более слайдов МОГУТ быть привязаны к одному §N,
    если все они начинаются внутри одного параграфа
  - Слайды идут в хронологическом порядке →
    §N следующего слайда ≥ §N предыдущего

Группировка нескольких слайдов на один параграф:
  Количество слайдов на один параграф не ограничено, но если их более трёх —
  привязка должна быть обоснована. Допустимые причины:
    ✅ Лектор показывает серию иллюстративных слайдов без смены темы речи
       (галерея примеров, несколько карт к одному тезису, сравнительные таблицы)
    ✅ Параграф длинный и тематически единый, слайды последовательно
       иллюстрируют его содержание
  Недопустимая причина:
    ❌ Слайды просто не удалось сопоставить с конкретным местом в тексте —
       это сигнал пересмотреть привязку, а не группировать по умолчанию


ШАГ 4. СФОРМИРОВАТЬ СПИСОК
----------------------------
Строки выводить в порядке возрастания §N.
Формат каждой строки:

  §N <figure><img K><figcaption>Заголовок</figcaption></figure>

Если несколько слайдов на один параграф — несколько строк подряд с одинаковым §N.

Если слайдов на один параграф более трёх — добавить строку-комментарий
с обоснованием после группы. Формат: строка начинается с #, с новой строки:

  §N <figure><img K1><figcaption>...</figcaption></figure>
  §N <figure><img K2><figcaption>...</figcaption></figure>
  §N <figure><img K3><figcaption>...</figcaption></figure>
  §N <figure><img K4><figcaption>...</figcaption></figure>
  # img K1–K4 на §N: галерея иллюстраций к теме X; параграф тематически единый


ПРОВЕРКИ ПЕРЕД ФИНАЛЬНОЙ ВЫДАЧЕЙ
==================================

□ ЧЕК 1. img N = номер страницы PDF
     Номер в <img K> должен совпадать с порядковым номером страницы в PDF.
     Проверить: все K от 1 до len(pages) присутствуют ровно по одному разу.

□ ЧЕК 2. figcaption = заголовок слайда
     Для каждого слайда:
       - Если явный заголовок ЕСТЬ → figcaption должен совпадать с ним дословно
         (допустимо незначительное сокращение, если заголовок очень длинный)
       - Если явного заголовка НЕТ → figcaption придуман; проверить, что он
         точно отражает содержание слайда

□ ЧЕК 3. Логическое соответствие §N теме слайда
     Прочитать первое предложение параграфа §N.
     Убедиться, что лектор там действительно начинает говорить о теме слайда K,
     а не заканчивает предыдущую тему.
     Если первое предложение §N ещё о другом → оценить весь параграф:
       - Если параграф целиком о предыдущей теме → сдвинуть на §N+1.
       - Если параграф начинается с хвостовой фразы, но основная его часть
         уже о теме слайда → оставить §N, не сдвигать.

□ ЧЕК 4. Количество figure-тегов = количество страниц PDF
     len(figure_list) == len(pdf.pages)
     Каждый слайд должен быть представлен ровно одним тегом.

□ ЧЕК 5. Все слайды присутствуют
     Множество {K} из всех <img K> == {1, 2, …, len(pdf.pages)}
     Нет пропущенных и нет дублирующихся номеров слайдов.

□ ЧЕК 6. Монотонность §N
     §N по всему списку должны быть нестрого возрастающими (каждый ≥ предыдущего).
     Все отклонения зафиксировать с пояснением (см. раздел «Непоследовательность
     слайдов»). Если отклонений нет — непоследовательности в лекции не обнаружено.


НЕПОСЛЕДОВАТЕЛЬНОСТЬ СЛАЙДОВ
------------------------------
В редких случаях лектор может показывать слайды не по порядку —
возвращаться к предыдущему или перепрыгивать вперёд.

Это ДОПУСТИМО и должно быть отражено в списке, если явно видно из текста.

Признаки реальной непоследовательности (надо зафиксировать):
  ✅ Лектор прямо говорит: «вернёмся к предыдущему слайду»,
     «давайте посмотрим снова на слайд про X», «я уже показывал этот слайд»
  ✅ Один и тот же слайд упоминается дважды в разных частях лекции
     с явным переключением («теперь следующий… нет, вернёмся»)
  ✅ §N следующего слайда МЕНЬШЕ §N предыдущего в списке
  ✅ Числовой порядок img в итоговом списке нарушен (слайд K стоит после
     слайда K+n, хотя явного маркера возврата нет) — такое нарушение тоже
     фиксируется комментарием: объяснить, почему слайд показан позже своего
     места в PDF

Как оформить: просто поставить img K в ту §N, где он реально показывается,
даже если это нарушает числовой порядок img.

Что НЕ является непоследовательностью (игнорировать):
  ❌ Лектор упоминает тему будущего слайда вскользь, не переключаясь
     («об этом подробнее позже…»)
  ❌ Первое упоминание ключевого слова слайда встречается раньше —
     но это просто забегание вперёд в речи
  ❌ Лектор делает короткое отступление и возвращается к той же теме

Проверка: если нет ни одного явного маркера навигации и §N всех слайдов
возрастают — непоследовательности нет, список в обычном порядке.

ПРИМЕР непоследовательности в тексте:
  §47 «давайте вернёмся к слайду про астику» → img 12 ставится на §47
  (несмотря на то, что img 12 уже стоял на §42)


ТИПИЧНЫЕ ОШИБКИ
---------------
- figcaption придуман там, где у слайда есть явный заголовок → нарушение чек 2
- Выбран заголовок по первой строке парсинга, а не по визуально выделенному
  элементу слайда → нарушение чек 2
- Два слайда с одинаковым заголовком оставлены без уточнения figcaption → чек 2
- Слайд привязан к параграфу, где лектор ещё заканчивает предыдущую тему → чек 3
- Слайд сдвинут на §N+1 формально (первое предложение не о теме), хотя большая
  часть §N уже о теме слайда → чек 3 (излишний сдвиг)
- Несколько «немых» вводных слайдов свалены в §0 без разбора по темам → шаг 1
- Технический параграф (возня с демонстрацией экрана) принят за содержательный
  и использован как §N → шаг 2
- Нарушение порядка img (слайд показан позже места в PDF) не прокомментировано → раздел «Непоследовательность»
- Более трёх слайдов на один параграф без обоснования → нарушение правила
  группировки (шаг 3)
- Пропущен слайд (нет <img K> для какого-то K) → чек 5
- Номера img не совпадают с номерами страниц PDF → чек 1
- Титульный слайд привязан не к §0 → нарушение правила титульного слайда
