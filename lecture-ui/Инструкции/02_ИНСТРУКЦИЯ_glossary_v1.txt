ИНСТРУКЦИЯ: генерация глоссария лекции на основе JSON-транскрипта
==================================================================

ВХОДНЫЕ ДАННЫЕ
--------------
- filename_dg.json     — транскрипт Deepgram (параграфы с таймкодами и текстом)
- filename_dg_toc.txt  — готовое оглавление (выход инструкции captions_list)

Оглавление используется для ориентации в структуре лекции,
но не ограничивает отбор терминов: термин может встречаться
в любом параграфе, независимо от границ разделов.


НАСТРОЙКИ ОТБОРА ТЕРМИНОВ
--------------------------
Перед запуском установить каждый флаг в ON или OFF:

  INCLUDE_SANSKRIT      = ON   # санскритские термины и их транслитерации
                                # даршана, сансара, атман, гуру, шастра…
  INCLUDE_CONCEPTS      = ON   # русские понятия, специфичные для курса
                                # инклюзивизм, ортодоксия, гетеродоксия…
  INCLUDE_PROPER_NAMES  = ON   # имена авторов, философов, мудрецов
                                # Патанджали, Шанкара, Капила…
  INCLUDE_TEXTS         = ON   # названия текстов, трактатов, сутр
                                # Йога-сутры, Веданта-сутра, Законы Ману…
  INCLUDE_SCHOOLS       = ON   # названия философских школ и традиций
                                # веданта, санкхья, локаята, адвайта…
  INCLUDE_GEOGRAPHY     = ON   # топонимы и географические названия
                                # Арьяварта, Синдху, Бхарата, Гандхара…

Флаг OFF полностью исключает категорию из выходного файла.
При ON=все категориях возможно пересечение (например, «санкхья» — и школа,
и санскритский термин): включать один раз, в наиболее точную категорию.


ВЫХОДНОЙ ФОРМАТ
---------------
Один термин на строку:

  термин [категория] {определение} §N §N …

где:
  - термин        — в той форме, в которой лектор его вводит
                    (санскритизмы курсивом не маркируются в txt-файле)
  - [категория]   — одна из шести меток (см. ниже)
  - {определение} — парафраз слов лектора, 1–2 предложения
  - §N            — все параграфы JSON, где термин существенно
                    упоминается или уточняется (через пробел)

Метки категорий:
Шесть базовых меток соответствуют флагам настроек. Каждая метка имеет
короткое английское имя для разметки файла и полное русское для отображения:

  [sanskrit]   — INCLUDE_SANSKRIT:      «Санскритские термины»
  [concept]    — INCLUDE_CONCEPTS:      «Понятия курса»
  [person]     — INCLUDE_PROPER_NAMES:  «Персоналии»
  [text]       — INCLUDE_TEXTS:         «Тексты»
  [school]     — INCLUDE_SCHOOLS:       «Философские школы»
  [geography]  — INCLUDE_GEOGRAPHY:     «География»

Если ни одна из базовых меток не подходит, LLM вводит новую метку —
короткое английское слово в квадратных скобках, например [deity], [dynasty].
Все нестандартные метки, использованные в данном файле, обязательно
перечисляются в заголовке файла в разделе «Дополнительные классы» —
с коротким английским именем и полным русским названием для отображения.

Пример объявления нестандартных меток:

  Дополнительные классы:
    [deity]   — «Боги и мифологические персонажи»
    [dynasty] — «Династии»

Пример:

  даршана [sanskrit] {система духовно-рациональных познаний, навсегда освобождающих человека от сансары; в отличие от греческой философии — инструментальна, заточена на результат} §34 §62 §65
  сансара [sanskrit] {цикл перерождений и страданий, из которого стремятся вырваться все индийские философские школы} §37 §68
  Арьяварта [geography] {историческое название северной части Индостана — от Гималаев до гор Виндхья; зафиксировано в Законах Ману} §7
  Шанкара [person] {философ, основоположник адвайта-веданты; основал четыре монастыря по кардинальным точкам Индии} §48 §49
  веданта [school] {одна из шести ортодоксальных школ; наряду с мимансой наиболее строго следует авторитету Вед} §42 §43
  инклюзивизм [concept] {стремление включать и ассимилировать то, что создавалось в других культурах; термин введён индологом Паулем Хакером} §14 §62

Порядок в выходном файле: алфавитный по термину.


ВЫХОДНОЙ ФАЙЛ
-------------
Сохранить результат в текстовый файл в кодировке UTF-8 с именем:
  <имя_входного_json>_glossary.txt
Пример: входной файл 01_vvip_dg.json → выходной файл 01_vvip_dg_glossary.txt

Заголовок файла должен содержать:
  - имя входного JSON
  - список флагов с их значениями ON/OFF
  - раздел «Дополнительные классы» — если в файле использованы нестандартные метки;
    если все метки базовые — раздел не нужен

Пример заголовка с дополнительными классами:

  ГЛОССАРИЙ: 02_веды_dg.json
  Флаги: INCLUDE_SANSKRIT=ON, INCLUDE_CONCEPTS=ON, INCLUDE_PROPER_NAMES=ON,
         INCLUDE_TEXTS=ON, INCLUDE_SCHOOLS=ON, INCLUDE_GEOGRAPHY=ON
  Дополнительные классы:
    [deity]   — «Боги и мифологические персонажи»
    [dynasty] — «Династии»


ШАГ 1. ИЗВЛЕЧЬ ПАРАГРАФЫ ИЗ JSON
----------------------------------
  import json

  data = json.load(open('filename_dg.json'))[0]
  paras = data['results']['channels'][0]['alternatives'][0]['paragraphs']['paragraphs']

  def fmt(s):
      t = int(s)
      return f'{t//3600:02d}:{(t%3600)//60:02d}:{t%60:02d}'

  for i, p in enumerate(paras):
      text = ' '.join(s['text'] for s in p['sentences'])
      print(f'§{i} [{fmt(p["start"])}] {text[:120]}')

(Тот же шаг, что и в инструкции по оглавлению — если оглавление уже готово,
параграфы уже извлечены и повторно извлекать не нужно.)


ШАГ 2. ПЕРВЫЙ ПРОХОД — ОТБОР КАНДИДАТОВ
-----------------------------------------
Читать параграфы насквозь. Для каждого параграфа отмечать:

  а) Термины, которые лектор явно вводит или объясняет:
       — даёт определение («X — это…», «X означает…»)
       — переводит с санскрита («слово X переводится как…»)
       — противопоставляет другому термину
       — называет впервые в лекции с пояснением

  б) Термины, которые лектор существенно уточняет или развивает
     (не просто упоминает вскользь, а добавляет новый смысловой аспект)

Фиксировать: термин + номер параграфа.
Простые упоминания без пояснения — не фиксировать.

Критерий «существенного упоминания»:
  — лектор добавляет новую характеристику термина
  — приводит пример, иллюстрирующий именно этот термин
  — уточняет границы понятия или противопоставляет смежному


ШАГ 3. ВТОРОЙ ПРОХОД — СОСТАВИТЬ ОПРЕДЕЛЕНИЯ
----------------------------------------------
Для каждого термина из списка кандидатов:

  1. Собрать все параграфы, где термин существенно упоминается.
  2. Составить определение строго как парафраз слов лектора:
       — не добавлять сведений из внешних источников
       — не использовать энциклопедические формулировки
       — если лектор дал чёткое определение — взять его за основу
       — если определение размазано по нескольким параграфам —
         синтезировать из них, не выходя за пределы сказанного
  3. Длина определения: 1–2 предложения, не более 40 слов.
  4. Зафиксировать все §N с существенными упоминаниями.

Правило первого параграфа:
  Первым в списке §N всегда стоит параграф, где термин введён
  или объяснён наиболее полно — даже если хронологически
  это не самое раннее упоминание.


ШАГ 4. ПРОВЕРКА ПОЛНОТЫ
------------------------
После составления списка терминов:

  1. Пройти по заголовкам оглавления (_toc.txt).
     Каждый заголовок, содержащий термин или имя собственное,
     должен быть отражён в глоссарии (если соответствующий флаг ON).
     Пример: заголовок «Анвикшики, дхарма, шастра, даршана» →
     все четыре термина должны быть в глоссарии.

  2. Убедиться, что ни один термин, явно введённый лектором
     (с определением или переводом), не пропущен.


ПРАВИЛА ТЕРМИНОЛОГИИ
---------------------
Форма термина:
  - Санскритские термины — в русской транслитерации, строчными:
    даршана, сансара, атман, брахман, гуру
  - Исключение — топонимы и географические названия: с заглавной буквы:
    Арьяварта, Синдху, Бхарата, Гандхара
  - Имена собственные (авторы, философы) — с заглавной:
    Патанджали, Шанкара, Капила, Харибхадра Сури
  - Названия текстов — с заглавной первого слова, курсив не маркируется в txt:
    Йога-сутры, Веданта-сутра, Законы Ману

Орфография (исправлять ошибки транскрипции):
  - санграха (не «самграха») — транслит. saṃgraha
  - аджи́вика (не «аджибика») — ударение на второй слог
  - Бхагавад-гита (не «Бхагавадгита»)
  - йога-сутры (не «йогасутры»)
  - Арьяварта (не «арьяварта») — топоним, с заглавной


ПРОВЕРКИ ПЕРЕД ФИНАЛЬНОЙ ВЫДАЧЕЙ
==================================

□ ЧЕК 1. Все термины из заголовков оглавления присутствуют
     Пройти по _toc.txt: каждый термин в заголовках h2/h3 →
     должен быть в глоссарии (при соответствующем флаге ON).

□ ЧЕК 2. Определения — только из слов лектора
     Для каждого определения: можно ли найти его основу
     в тексте параграфов? Если нет → переформулировать.
     Энциклопедические формулировки недопустимы.

□ ЧЕК 3. §N корректны
     Для каждого термина: все указанные §N действительно содержат
     существенное упоминание термина. Проходные упоминания — убрать.

□ ЧЕК 4. Первый §N — параграф с основным определением
     Первый в списке §N — тот, где термин объяснён наиболее полно,
     а не просто упомянут первым по хронологии.

□ ЧЕК 5. Нет дублей
     Один и тот же термин не должен встречаться дважды.
     Варианты одного термина (санкхья / Санкхья-карики) —
     это разные термины, держать раздельно.

□ ЧЕК 6. Флаги соблюдены
     Термины из категорий с флагом OFF отсутствуют в выходном файле.

□ ЧЕК 6а. Нестандартные метки объявлены в заголовке
     Если в файле использована метка, отсутствующая в базовом списке,
     она должна быть перечислена и объяснена в разделе «Дополнительные классы»
     заголовка файла. Необъявленная нестандартная метка — ошибка.

□ ЧЕК 7. Алфавитный порядок
     Список отсортирован по алфавиту (строчные и заглавные
     сортируются вместе, по первой букве).

□ ЧЕК 8. Длина определений
     Ни одно определение не превышает 40 слов.
     Слишком короткое (< 5 слов) — скорее всего неполное, уточнить.

□ ЧЕК 9. Орфография терминов
     Санскритские термины проверены по правилам терминологии.
     Топонимы — с заглавной буквы. Имена — с заглавной буквы.


ТИПИЧНЫЕ ОШИБКИ
---------------
- Определение взято из энциклопедии, а не из слов лектора → чек 2
- Термин из заголовка оглавления пропущен в глоссарии → чек 1
- §N указан для проходного упоминания, а не существенного → чек 3
- Первым стоит §N с ранним упоминанием, а не с основным определением → чек 4
- Топоним написан строчными (арьяварта вместо Арьяварта) → терминология
- «самграха» вместо «санграха» → терминология
- Термин из категории с флагом OFF попал в список → чек 6
- Нестандартная метка использована, но не объявлена в заголовке файла → чек 6а
- Для термина выбрана ближайшая базовая метка, хотя он явно требует нестандартной → чек 6а
- Два варианта одного термина объединены в одну строку → чек 5
- Список не отсортирован по алфавиту → чек 7
- Определение длиннее 40 слов → чек 8
